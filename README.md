## Profile Overview

Soy **Ingeniero de Sistemas** enfocado en el desarrollo de **Infraestructura para Inteligencia Artificial** y **Arquitectura de Software** de alto rendimiento. Mi trabajo se centra en eliminar la "caja negra" de la IA, creando herramientas que permiten auditoría, trazabilidad y ejecución tanto en Cloud y local de forma eficiente y segura, usando protocolos como HTTPS, encriptación entre otros métodos.

Mi enfoque técnico se divide en tres pilares:
1.  **AI Infrastructure:** Desarrollo de motores de bases de datos vectoriales y SDKs en Rust | Python.
2.  **Data-Driven Auditing:** Transformación de la auditoría de código en métricas visuales (JSON -> Radar Charts) para entornos ERP.
3.  **Generative AI Pipeline:** Ingeniería de nodos personalizados para flujos de trabajo de producción en ComfyUI.

## Featured Engineering

### <img src="https://raw.githubusercontent.com/simple-icons/simple-icons/develop/icons/rust.svg" width="18" height="18" /> Luma Project (Formerly RustKissVDB)
> *High-performance Vector Database & SDK*

Sistema de almacenamiento diseñado desde cero en **Rust** para gestión eficiente de embeddings y datos relacionales.
* **Luma:** Motor de base de datos multimodal que implementa índices vectoriales nativos y gestión de memoria segura sin *garbage collection*.
* **LangChain-Luma SDK:** Librería de integración para Python que orquesta el levantamiento de un **servidor binario** autónomo, permitiendo a los desarrolladores consumir la base de datos como un servicio local sin configuración compleja.

### <img src="https://raw.githubusercontent.com/simple-icons/simple-icons/develop/icons/json.svg" width="18" height="18" /> NetSuite Auditor Architecture
> *Quantitative Code Analysis System*

Sistema de auditoría para **Oracle NetSuite (SuiteScript 1.0, 2.0 y 2.1)** que reemplaza la revisión manual subjetiva por análisis métrico.
* **System Prompt Engineering:** Uso de definiciones estrictas (`auditor.md`) para analizar estáticamente el código.
* **Models Open Weight** Uso de modelos de código abierto como OpenAI OSS 120B para garantizar un estandar y calidad de auditoría.
* **JSON Output & Visualization:** El sistema no genera texto, sino un objeto **JSON** estructurado que alimenta un **Diagrama de Radar Octagonal**. Esto permite visualizar deuda técnica en 8 dimensiones (Seguridad, Modularidad, Performance, Gobernanza, etc.) de un solo vistazo, facilitando la toma de decisiones.

### <img src="https://raw.githubusercontent.com/simple-icons/simple-icons/develop/icons/artstation.svg" width="18" height="18" /> ComfyUI Custom Nodes
> *Production-Grade Generative AI*

Desarrollo de nodos personalizados en Python para extender la lógica de **ComfyUI**.
* Control preciso de latentes y pre-procesamiento de inputs.
* Integración de APIs externas directamente en el flujo de generación de imágenes.
